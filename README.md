# Regulated AI Orchestration Using Attestix

Internship project demonstrating integration of the Attestix Protocol as a Zero-Trust security middleware across four multi-agent AI systems built with different orchestration frameworks. Authored as part of a B.Tech project at NIT Patna and as a work product at VibeTensor Private Limited (Attestix is VibeTensor's open-source library).

Each domain runs two pipelines side-by-side:
- `agents/`   - unregulated baseline (no security layer)
- `c_agents/` - regulated pipeline (Attestix-secured)

## What Attestix adds

All four regulated pipelines use a single shared `AttestixManager` instance from `shared/attestix_client.py`. Each domain configures it with a 9-line factory call shim under `<domain>/c_agents/attestix_client.py`.

| Pillar              | Attestix Service     | What it does                                                       |
|---------------------|----------------------|--------------------------------------------------------------------|
| Agent Identity      | `IdentityService`    | Ed25519 DID + Universal Agent Identity Token per agent             |
| Access Control      | `DelegationService`  | UCAN-style JWT token gates each inter-agent handoff                |
| Role Certification  | `CredentialService`  | W3C Verifiable Credential signed per agent role (Ed25519Sig2020)   |
| Audit Logging       | `ProvenanceService`  | SHA-256 Merkle hash-chained tamper-evident log + Article 10/11     |
| Compliance          | `ComplianceService`  | EU AI Act Annex III profile; gate blocks unregistered agents       |
| Blockchain Anchor   | `BlockchainService`  | Merkle root anchored to Base Sepolia via EAS (optional)            |

Local data stored under `~/.attestix/`. No external database required.

---

## Threat model and limitations (read this before deployment)

This is a proof-of-concept. It demonstrates the integration shape and exercises the cryptographic primitives end-to-end. It does NOT yet:

- Isolate per-agent signing keys. The orchestrator process holds a singleton with implicit access to all agent identities. Production deployment requires per-agent key material managed out-of-process.
- Encrypt audit storage at rest. `~/.attestix/*.json` is plaintext.
- Exercise multi-hop UCAN attenuation chains. Each domain mints flat single-hop delegations from one principal to another.

Future Works (Report Chapter 7) enumerates the migration path.
