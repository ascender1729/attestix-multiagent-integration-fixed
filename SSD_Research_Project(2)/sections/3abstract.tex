\begin{center}
{\bf \Large ABSTRACT}
\end{center}
\vspace{0.05in}

\setlength{\parindent}{15pt}
\setlength{\parskip}{0.4em}

Autonomous AI agents are being deployed in regulated sectors such as healthcare, finance, law, and environmental governance. Existing multi-agent frameworks (LangChain, CrewAI, LangGraph, Semantic Kernel, OpenAI Agents SDK) were built for capability rather than compliance: they provide no built-in mechanisms for verifying agent identity, controlling data access, or maintaining tamper-proof records of decisions.

This project integrates the Attestix Protocol, a pip-installable Python security library, as a non-invasive middleware across four domain simulations: an AI-Assisted Medical Panel, an Algorithmic Hedge Fund, a Global ESG Supply Chain Auditor, and an Autonomous Legal Advisory Panel. Each domain uses a different mix of orchestration frameworks, demonstrating that the security layer is framework-agnostic.

Five security pillars are exercised end-to-end: Ed25519-backed Decentralised Identifiers (DIDs); UCAN delegation tokens with cryptographic verification; W3C Verifiable Credentials for role certification; SHA-256 Merkle provenance chains for tamper-evident decision logging; and EU AI Act compliance profiles aligned to Annex III risk categories. An optional extension anchors the Merkle root to Base Sepolia via the Ethereum Attestation Service (live attestation UID: \texttt{0xb2cdbd85...}).

The integration is validated by a 13-case automated test suite (13/13 passing) and a 20-fixture golden-prompt evaluation harness with seven adversarial safety probes; the Hospital pipeline achieves an aggregate evaluation score of 0.890 (above the 0.80 regression bar) and five of seven safety probes pass outright. The pipelines are provider-agnostic at the inference layer (Groq \texttt{llama-3.3-70b-versatile} or AWS Bedrock \texttt{us.meta.llama3-3-70b-instruct-v1:0} via a single factory), and an OWASP LLM01 prompt-injection mitigation layer is applied at every external-input boundary.
